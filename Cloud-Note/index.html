Typora+PicGo+Girhub+坚果云创建云笔记

介绍如何搭建一个自己的MarkDown云笔记

[toc]

创建图床（以GitHub为例）

1. 注册并登录GitHub账号，点击新建一个公共的仓库

2. 在账户设置中，获取一个TOKEN,保存好生成的TOKEN备用

注意Token只在第一次创建后显示，刷新页面就消失，要即使保存

安装PicGo

PicGo: 一个用于快速上传图片并获取图片 URL 链接的工具。基于electron-vue开发，支持macOS,Windows,Linux。支持的图床多样，支持自定义插件。

1. 安装Node.js(版本大于8)

如已经确认环境符合要求，跳过此步骤
确认node版本，下载安装：

windows环境下为例：按下“win+R”打开“运行”，输入“cmd”打开命令行，输入node -v

3. 配置图床，添加插件

以GitHub图床配置为例

添加插件
这里建议添加Github-plus，它可以帮助你同步删除本地相册和远程仓库的图片

踩坑

1.PicGo上传图片失败

由于GitHub是国外的平台，国内对外网访问有限制，所以经常由于网络原因导致上传出错。同理，在GitHub图床的图片，在笔记打开是可能出现加载缓慢或出错的情况。如果介意这点，请选择国内的图床（Gitee对大于1M的图片外链访问有限制，需要登录账号）。

2.PicGo的相册显示异常

本人遇到两种情况：一是上传成功，远程仓库有图片，相册有图片但无法预览；二是仓库上传成功，但仓库没有该图片。
出现这种原因多种，尤其注意上传的图片文件名不要包含特殊字符，保险起见一律采用纯英文和数字，或在PicGo中设置“上传时时间戳重命名”。

3.Server端口占用导致失败

请在设置中使用推荐的默认端口，否则极容易上传失败。

4.在PicGo在线安装插件始终显示安装中

重启软件，在设置中更换插件的镜像源 https://registry.npm.taobao.org

或者打开命令行 输入命令
npm config set registry https://registry.npm.taobao.org

5.无法同步删除远程库的图片

正确安装，github-plus插件，确认相关配置信息填写是否正确
记得设置插件后，把github-plus设置为默认

6.插入图床图片，显示缓慢

这是因为github访问缓慢，我们可以在图床的配置中填入自定义域名，这里采用的时jsdliver的加速域名，能帮助我们更快加载GitHub图床图片。
https://cdn.jsdelivr.net/gh/用户名/仓库名

安装Typora

Typora是一款轻量级的PC端MarkDown写作工具，支持自定义主题。不同于其他主流的MarkDown编辑器采用代码模式和渲染效果分屏显示，Typora采用即时渲染，做到了“所见即所得”。此外，专注模式/打字机模式、大纲、文件树等功能也更加便于编写和查看。

下载地址
主题下载

Typora中设置插入图片自动上传

打开Typora偏好设置，点击图像，选择插入图像自动上传，勾选应用到本地和网络图片。上传服务选择PicGo(app)，填写PicGo安装路径。
完成后，打开PicGo，点击验证设置，查看是否能够成功上传。

到此，所有的工作完成。在用Typora编写MarkDown的时候，插入图片都会自动上传保存，即使移动和分享文档也能够加载图片。

坚果云网盘云同步

最后为了能够在各个设备随时查看笔记，使用网盘对文件进行同步。
这里选择的是坚果云，它支持自动同步文件夹、支持MarkDown、上传下载不限速，每月有流量限制，但对于编写MarkDown来说足够了，图片用图床保存，文件本体很小。
